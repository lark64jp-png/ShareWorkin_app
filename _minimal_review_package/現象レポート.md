# ShareWorkin 現象レポート

対象コミット:
- `2f06f4c Retry notification dispatch after trusted certificate refresh`

対象インストーラー:
- `H:\ShareWorkin_app\ShareWorkin_v1.21_install.exe`

現在の事実:
- 本機 `Win11` では `notifycert.dat` が作成済みで、起動ログにも保存済み通知TLS証明書の再読込が出る
- それでも `東芝 -> Win11` のファイル送信では、Win11 側は正規交流通知ではなく未照合受信になる
- Win11 側ログでは受信時刻付近に `HandleClientAsync error: The decryption operation failed, see inner exception.` が出る
- Win11 側ログには今回時刻の `InteractionEventNotice accepted` が出ていない
- 東芝側更新履歴では `交流通知送達失敗` が出る
- 通知入力テキストは、正規通知が届いていないため Win11 側に表示されない

直近の善処修正:
- 通知TLS証明書を `notifycert.dat` へ永続化し、再起動後も再利用できるようにした
- インストーラー作成時の版情報解決を修正した
- 通知送信時に証明書不一致が出た場合、共有接続で同一相手確認済みなら一度だけ証明書情報を更新して再送する処理を追加した

直近テスト結果:
- Win11 起動表示は `v1.21+a18f259` までは確認済み
- その後のテストでは、Win11 側が `未照合受信`、東芝側が `交流通知送達失敗`
- メッセージ本文は未表示

再現手順:
1. 東芝側で `Win11` の共有を開く
2. 通知テキストを入力して `Win11` 宛てにファイルを配置する
3. Win11 側の更新履歴・通知履歴・トースト通知を確認する
4. 東芝側の更新履歴を確認する

期待:
- Win11 側で正規交流通知として受信される
- 通知本文に入力テキストが出る
- 東芝側で `交流通知送達失敗` が出ない

実際:
- Win11 側では未照合受信
- 通知本文なし
- 東芝側では交流通知送達失敗
