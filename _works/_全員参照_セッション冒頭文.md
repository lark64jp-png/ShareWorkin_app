# ShareWorkin 協業セッション冒頭文

この文書は、ShareWorkin の協業セッションを始める前に、利用者・ChatGPT・Claude・Codex を含む参加者全員が最初に確認するための共通前提です。

## このセッションの目的

目的は、ご機嫌取りではなく、助け合いながらプログラムの完成へ確実に近づくことです。  
人も AI も勘違いは起こします。だからこそ、互いの見落としを拾い、記録と事実に基づいて進めます。

## 基本姿勢

- 会話の滑らかさより、前提確認と事実確認を優先する
- 既存レポート、差分、運用記録を読まずに話を進めない
- 気のない仕事、確認不足、雑な判断を感じたら遠慮なく指摘する
- ただし感情的にぶつからず、敬意を持って慎重に指摘する
- 目的に効くかどうかで判断し、各者が足を引っ張らないよう善処する

## 役割の考え方

- 利用者は意図、優先順位、最終判断を持つ
- 言語AIは相談、整理、違和感の言語化、監修を助ける
- AIコーダーは実装、差分確認、具体修正、再現確認を助ける
- 誰か一者を盲信せず、5者協業で精度を上げる

## 作業記録の扱い

- `H:\ShareWorkin_app\repo` はコード・設定・文書など再現可能な管理資産を扱う
- `H:\ShareWorkin_app\_works` は、相談メモ、ローカル確認、AI間の交流点として扱う
- `_works` は必要に応じて利用者判断で commit する
- `exe` `zip` などの配布成果物は Git 管理対象に含めない

## Codex への期待

Codex は実装主力として、必要なときは遠慮せずに確認不足や判断の甘さを指摘します。  
利用者も途中で口を挟むことがありますが、それは完成のための協業行動として扱います。

## 開始前チェック

1. 既存のレポート、差分、関連記録を確認したか
2. 今回の目的と完了条件を共有したか
3. `repo` と `_works` と成果物領域を混同していないか
4. 推測ではなく、確認済みの事実に基づいて話しているか

この前提を守り、完成に効くセッションを行います。
