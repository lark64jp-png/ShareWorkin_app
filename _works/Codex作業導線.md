# Codex作業導線

## 目的

`_works` を整理したあとでも、Codex が迷わず同じ前提で起動し、最初に読む文書の場所をすぐ把握できるようにするための案内。
Windows の「所有者」を操作すること自体が主目的ではなく、Codex から見て必要な場所に触れられない状態を減らし、作業開始時の迷いを減らすことを優先する。

## 起動入口

- `psスクリプト/__Codex権限.bat`
  通常運用の入口。`Start-Codex-ShareWorkin.ps1` を呼び出し、承認方針は `on-request`。
- `psスクリプト/__Codex権限_NoPrompt.bat`
  強い権限で一気に動かす入口。`Start-Codex-ShareWorkin-NoPrompt.ps1` を呼び出し、承認とサンドボックスをバイパスする。

## 起動スクリプトの意図

- 作業ルートを `H:\ShareWorkin_app` に固定する
- `.claude` を追加対象にして、コンテンツ文脈を読める前提で起動する
- `--search` を有効にして、必要時のWeb参照を使えるようにする
- 毎回の起動条件をそろえて、Codex が環境差分で迷わないようにする
- Windows のオーナー概念そのものを調整するより、Codex が実務上読み書きできる範囲を安定させることを重視する

## 協業方針

- Claude との協業は前提に含むが、作業の主系は Codex として扱う
- `.claude` は Codex が文脈把握のために参照してよい
- `.codex` は Codex の記録領域として尊重し、Claude 側は参照にとどめる
- 判断に迷う構成変更をするときは、Codex が先に迷わない導線を残してから周辺文書を更新する
