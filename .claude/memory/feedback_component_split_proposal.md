---
name: コンポーネント分割提案の却下は後で直しミスになることがある
description: 重複ロジックのコンポーネント統一を提案→却下→後に直しミス発生、という経緯の記録
type: feedback
---

スキャンロジックが FriendsWindow と UserListWindow で二重実装になっていた際、「同一コンポーネントブロックに統一すべき」と提案したが「場所が少ないので不要」と却下された。その結果、一方を修正した際に他方が修正漏れとなり、スキャン結果の不整合バグを引き起こした。

**Why:** 重複コードは「今は少ない」としても、修正が片方だけに入るリスクがある。提案を却下する際はそのリスクを西村さんに伝えておく価値がある。

**How to apply:** 同様の重複ロジックを見つけたとき、却下されても「修正漏れリスク」として一言添えて記録しておく。ただし実装は西村さんの判断に従う。
