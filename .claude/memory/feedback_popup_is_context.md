---
name: ポップアップ自体がコンテキスト
description: 吹き出しの矢印が元の行を指している時点で「対象」は自明。タイトル・対象名・説明ラベルなど「親画面に書いてある情報」をポップアップ内で繰り返さない。
type: feedback
---

吹き出し型ポップアップは、**矢印が指している対象 = 操作対象** という構造そのものでコンテキストを伝える。だから:

- ❌ 「共有許可」のようなタイトルを別行で出す
- ❌ 「対象: Win11と東芝_R」のように親画面で見えているファイル/フォルダー名を再表示する
- ❌ 「このフォルダーは…」のような説明文を加える
- ❌ ボタン操作と冗長な機能（クリアが既にあるのに右向き全クリア矢印を別途置く 等）

すべて「ポップアップなのに普通のダイアログのように扱っている」サイン。

**Why:**
- 2026-05-20 セッション。許可指定ポップアップの再設計で、毎回タイトル行・対象名・列見出し・ヒント文・移動矢印を「説明として親切」と思い込んで足してしまった
- 西村さんから「ポップアップ故のメリットをもう少し理解してください」「ラフから読み取れるはず」と複数回指摘
- 吹き出しの形そのものが「どの行のことか」を物理的に示しているので、文字で繰り返すのは情報の重複でしかない

**How to apply:**
- ポップアップ/吹き出し系の実装を始める前に、まず「親画面から見えている情報は何か」を列挙する。それを popup 内に書かない
- 親画面の文脈で意味が通じる短い名詞だけで動作を構成する（例: 「全員」「クリア」「非ユーザー」だけで操作は完結する。「許可リストをクリアします」とは書かない）
- 既にある操作と機能が重複するUI要素（クリアと右向き全クリア矢印など）は、片方しか入れない
- ラフ/スケッチに無い要素を「親切のつもり」で足さない。スケッチに描かれているもの = 必要なもの、描かれていないもの = 意図的な省略
