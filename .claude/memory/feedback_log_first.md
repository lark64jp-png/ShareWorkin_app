description: ログはテストと修正の橋。ログが取れていない=手がかりなしで直している状態。コーディングより先に確認すべきもの
type: feedback
---
テストで問題が起きたとき、ログを真っ先に確認する。ログに記録されていない失敗は、推測で直すしかなくなる。

**Why:** タイムアウトエラーが画面に出ているのに、ログに痕跡が一切なかった（`RequestInviteCodeAsync` の `OperationCanceledException` パスにログがなかった）。これにより原因特定が大幅に遅れた。

**How to apply:**
- エラー・失敗のすべてのパスに必ずログを入れる。特に `catch` ブロック
- ログに書いていない `return` は「黒い穴」。何が起きたか後から追えない
- 新機能・修正を入れるとき「この処理が失敗したとき、ログに何が残るか」を必ず確認する
- テスト開始前にログファイルの場所を把握しておく（`C:\MyApps\ShareWorkin\logs\swk_yyyy-MM-dd.log`）
- 画面に出ているエラーメッセージは事実として受け取る。「OKを押したか」等を疑う前にログで確認
