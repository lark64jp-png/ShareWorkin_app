## 構造変更：FriendsWindow（ユーザー情報ダイアログ）
- **二列レイアウト** に変更：左＝プロフィール／右＝許可状況・履歴2種（**右列は空枠先置き**、中身は後の抽出仕事）
- 上段右に [削除][確定]：新規モードでは [削除] をグレーアウト（IsEnabled=False、非表示ではない）
- 中段：左に「スキャンモード切替＋再スキャン」（前画面UserListWindowのミラー、現[再読み込み]はこれに合流）／右に [アイコン][閉じる]
- [フォルダーを開く] 廃止（もともと場違い）
- [キャンセル] → [閉じる] に統合
- 候補リスト：登録不可な行は IsSelectable=False でグレーダウン＋選択不可

## 機能追加：アイコン
- `Friend.IconKey` 追加（`""` ／ `lib:<name>` ／ `user:<filename>`）
- `IconService` 新設：`<AppHomeDir>\icons\library\` ／ `<AppHomeDir>\icons\custom\` を管理
- ライブラリ16種：thumb / hood / cup / ninja / rose / bowl / moon / purse / face_a〜face_h（西村さん作成のクリーンPNG 100×100、`Assets\icons\library\` 同梱、csproj で出力先 `icons\library\` にコピー）
- `IconPickerWindow`：4×4ライブラリ（56×56セル正方形ロック、画像40×40）／プレビュー兼ドロップゾーン100×100（保存サイズに合わせジャストフィット）／[参照…][クリア] と [確定][閉じる] をフッター集約／素材サイト3リンク（icooon-mono / icon-z / icon-pit）
- [参照]はWindows標準OpenFileDialog（JPG/JPEG/PNG）。ドロップ＆ドロップも同経路
- ステージング方式：ライブラリ／参照／ドロップ／クリアはプレビューにステージのみ。[確定]で初めて実体化（custom フォルダーへコピー、IconKey 確定）。[閉じる] でキャンセルすればファイルは残らない
- 新規モードでカスタム画像を入れる場合、`_pending_<guid>` の仮IDで保存→確定時に `friendId` で改名
